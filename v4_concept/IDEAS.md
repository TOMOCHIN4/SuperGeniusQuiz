# 新アプリ構想メモ

> 取り留めもなく出てきたアイデアをまとめていくドキュメント

---

## コンセプト

**さくさく進む4択クイズ型学習アプリ**

- 軽快さと達成感が重要（利用回数を増やすため）
- 同じ知識でも複数の問い方を用意 → 応用力を広げる

---

## 画面フロー

```
[起動画面]
    ↓ ID/パスワードでログイン
[ホーム画面]
    ├── おおまかな実績表示
    └── メニュー
         ├── PLAY → 教科選択へ
         ├── 詳細な実績分析
         └── （その他？）
              ↓
[教科選択]
    ├── 国語 → [ジャンル選択] → クイズ
    ├── 算数 → [ジャンル選択] → クイズ
    ├── 理科 → [ジャンル選択] → クイズ
    ├── 社会 → [ジャンル選択] → クイズ
    └── 全教科 → クイズ
              ↓
[クイズ画面]
    10問/教科、制限時間あり
```

## 制限時間（教科別）

| 教科 | 時間 | 理由 |
|------|------|------|
| 算数 | 長め（3〜5分？） | 計算に時間がかかる |
| 理科 | やや長め（2〜3分？） | 計算問題がある場合 |
| 国語 | 1〜2分 | そんなに時間かからない |
| 社会 | 1〜2分 | そんなに時間かからない |

## 問題設計

**同じ知識を複数の問い方で出題**

例：「鎌倉幕府の成立年」
- パターンA: 「鎌倉幕府が成立したのは何年？」
- パターンB: 「1185年に起きた出来事は？」
- パターンC: 「源頼朝が幕府を開いた場所と年は？」

→ 知識の定着 + 応用力UP

---

## 重要な価値観

1. **軽快さ** - サクサク動く、待たない
2. **達成感** - やった感がある、続けたくなる
3. **シンプル** - 迷わないUI
4. **拡張しやすい** - コンテンツを簡単に追加できる

---

## 設計方針：コンテンツ拡張性

**初期リリースは早く、コンテンツは後から追加**

```
初期リリース
├── 国語（基本ジャンルのみ）
├── 算数（基本ジャンルのみ）
├── 理科（基本ジャンルのみ）
└── 社会（基本ジャンルのみ）

↓ 後から追加

v1.1: 算数に「速さ」ジャンル追加
v1.2: 社会に「歴史・江戸時代」追加
v1.3: ...
```

**コンテンツ追加 = ファイル追加だけで完了**する設計が理想

例：
```
questions/
├── math/
│   ├── basic.json      ← 初期
│   ├── speed.json      ← 後で追加
│   └── ratio.json      ← 後で追加
├── japanese/
│   └── ...
└── ...
```

→ JSONファイルを追加するだけで新ジャンルが使えるようにしたい

## フロントエンド方針

- アニメーションは少なく、シンプル
- **軽快さを最優先**
- 派手さより速さ

---

## データ管理方針

**Google Sheets でシンプルに**

GAS + HTML で完結させるなら → データは Google Sheets

- 問題集: スプレッドシート（シートやタブで教科・ジャンル分け）
- ユーザー・実績: スプレッドシート（当面3人程度）

**メリット**
- スプレッドシート上で直接編集・追加できる
- 行を追加するだけでコンテンツ追加
- コード変更不要
- バックアップ・履歴も自動

---

## 運用フロー

**問題作成はオフライン（手動 + AI）**

```
[開発者]
    ↓ ChatGPT等で問題作成
[スプレッドシートに追加]
    ↓ 手動でコピペ
[アプリに即反映]
```

**リリースと並行してコンテンツ拡充**

```
Week 1: 初期コンテンツでリリース（各教科 基本問題）
        ↓ ユーザーが学習中
Week 2: 算数「速さ」追加 → 新着として表示
        ↓ ユーザーが学習中
Week 3: 社会「江戸時代」追加 → 新着として表示
        ...
```

**新着コンテンツだけで学習も可能**

- 「最近追加された問題」から入れば新しいのだけ解ける
- 既存ユーザーも飽きずに続けられる

**UI上の工夫**
- 「最近追加された問題」リストを表示 → ユーザーが新コンテンツに気づきやすい
- 教科 → テーマごとにわかりやすいボタン配置

## 技術選択（こだわりなし）

**要件**
- 当面3人のユーザーが使えれば十分
- コンテンツ追加が簡単
- 使用制限が気にならない

**有力候補：GAS + Google Sheets + HTML**

```
[Google Sheets]        [GAS]           [HTML/CSS/JS]
 問題集                  API            フロントエンド
 ユーザー        ←→    ロジック    ←→   UI
 実績
```

- Googleだけで完結
- スプレッドシートで問題追加が簡単
- 3人程度なら使用制限は問題なさそう
